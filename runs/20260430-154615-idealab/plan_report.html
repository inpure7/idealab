IdeaLab · 아이디어가 떠오르는 속도만큼 빠르게 정리·목업·추적하는 1인 창업자의 귀여운 아이디어 연구실
요약 특허 상세 시장/전략 상세 기획서 QA MVP 목업
기획서(무엇을 했는지)
  • 제품명: IdeaLab
  • 타겟: 바이브코딩 활용 1인 창업자, 다수의 마이크로 SaaS를 병렬 운영하는 솔로프리너, 사이드프로젝트 인디해커
  • 문제: 아이디어 발생 속도 > 구현 속도로 인한 병목. 수십~수백개 아이디어가 분산되고, 어떤 서비스를 먼저 키울지 데이터 근거 없이 감으로 결정하게 됨
  • 솔루션: 아이디어 캡처 → AI 목업 자동생성 → MVP 버전별 운영 → 서비스별 트래픽 통합 대시보드로 우선순위 자동 추천까지 이어지는 원스톱 아이디어 포트폴리오 플랫폼
  • 비즈니스 모델: Freemium SaaS. 무료 플랜으로 1인 창업자를 유입 후 아이디어 수, AI 크레딧, 트래픽 분석 깊이 기반으로 자연스러운 업셀링. AI 기능은 크레딧 종량제 병행.
MVP 기능
  • 아이디어 카드 생성 및 상태관리 (아이디어→목업→MVP→운영 파이프라인)
  • AI 기반 기초 시장분석 자동생성 (경쟁사, 시장규모, 차별점 요약)
  • 와이어프레임/목업 자동 생성 (텍스트 설명 → UI 스케치)
  • 서비스별 버전 트래킹 (v1, v2... 릴리즈 노트 + 상태 관리)
  • 트래픽 통합 대시보드 (서비스별 DAU/MAU 한눈에 비교)
  • 우선순위 자동 추천 (트래픽+성장률 기반 집중 대상 알림)
  • 아이디어 포트폴리오 뷰 (연구실 책장 컨셉 전체 조망)
  • 공개/비공개 쇼케이스 (아이디어 공개 후 얼리유저 모집)
기술 스택
frontend: Next.js + Tailwind CSS (귀여운 연구실 디자인 시스템)backend: FastAPI + Celery (비동기 AI 처리)db: PostgreSQL + Redis (트래픽 집계 캐싱)ai: OpenAI API (목업 생성, 시장 분석)analytics: 자체 트래픽 수집 SDK + Clickhouse
수익 구조
  • Free: $0 (아이디어 5개, 목업 3회/월, 트래픽 연동 1개)
  • Pro: $12/월 (무제한 아이디어, 트래픽 통합 대시보드, AI 분석 10회/월)
  • Studio: $39/월 (팀 공유, 무제한 AI, 클라이언트 포트폴리오, 우선지원)
  • AI 크레딧 종량제: 목업·시장분석 추가 사용 $0.5/회
  • 스폰서십: 바이브코딩 도구(Cursor, Replit 등) 파트너 노출
차별점
  • 아이디어 인큐베이팅→목업→MVP→버전운영→트래픽분석 원스톱 파이프라인 (경쟁사 전무)
  • 서비스별 실사용 트래픽 집계 기반 개발 우선순위 자동 추천 피드백 루프
  • 수십~수백개 아이디어 버전별 병렬 관리 포트폴리오 뷰 (연구실 책장 UI)
  • 귀엽고 따뜻한 연구실 브랜딩으로 딱딱한 PM툴 대비 감성적 창업 경험 제공
  • 바이브코딩 1인 창업자 특화 컨텍스트 (Notion·Linear는 팀/엔터프라이즈 중심)
  • AI 시장분석 자동화로 아이디어 검증 초기 리서치 시간 90% 단축
리스크
  • 1인 창업자의 낮은 지불의사(WTP)로 유료 전환율 확보 어려움 → 무료 플랜 바이럴로 볼륨 확보 후 전환 유도
  • Notion·Linear 등 대형 플랫폼이 1인 창업자 특화 기능 추가 시 빠른 시장 잠식 가능
  • 바이브코딩 도구(Cursor, Replit) 자체가 아이디어 관리 기능 내재화할 위험
  • 아이디어 데이터 보안·유출 민감도 높아 신뢰 구축에 초기 비용 소요
  • 기능 범위 과대로 MVP 완성도 vs 출시 속도 트레이드오프 관리 실패 위험
  • 트래픽 수집 SDK 각 서비스에 설치해야 하는 온보딩 마찰 → 설치 간소화 필수
마일스톤
  • Week 1 — 핵심 DB 스키마 설계 (아이디어, 버전, 트래픽 모델) + 디자인 시스템 구축 (연구실 컨셉 컴포넌트)
  • Week 2 — 아이디어 카드 CRUD + 파이프라인 상태 관리 (아이디어→목업→MVP→운영) 구현
  • Week 3 — OpenAI 연동 AI 시장분석 자동생성 + 기초 목업 스케치 생성 기능 완성
  • Week 4 — 트래픽 수집 SDK 개발 + 서비스별 통합 대시보드 + 버전 트래킹 기능 완성
  • Week 5 — 포트폴리오 뷰 (연구실 책장 UI) + 우선순위 자동추천 알고리즘 + 공개 쇼케이스 페이지
  • Week 6 — Product Hunt 런칭 + 인디해커스 커뮤니티 공개 + 얼리어답터 100명 확보 및 피드백 수집
